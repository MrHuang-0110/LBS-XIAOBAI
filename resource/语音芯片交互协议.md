# 小白-Ai MCU ↔ ASRPRO UART 交互协议 v0.4

- 更新日期：2026-07-03（v0.4：取消 `\r\n` 帧尾，改为 UART IDLE 分帧）
- 物理层：UART 115200 8N1 全双工
- 目标 MCU：PY32F030K28U6TR
- 语音芯片：ASRPRO
- 最大帧长：**32 字节**

---

## 一、帧格式

ASCII 文本，**无任何显式帧尾**。收方靠 UART IDLE 中断（数据静默超过 1 字符时间）分帧。

**MCU → ASRPRO：** `=` 作为 tag 与数值分隔
```
<tag>[=<dec>]
```

**ASRPRO → MCU：** `:` 作为 tag 与数值分隔
```
<tag>[:<dec>]
```

- `tag`：全小写英文字母，长度 ≤ 8
- `dec`：十进制数值字符串（可 `1` / `01` / `10` 都接受）
- 无 `\r\n` 也无其它结束符

**关键约束（发送方必须遵守）：**
- 每帧发完至少空 **1 个字符时间（≈100µs @115200）** 才能开始下一帧，让接收方 IDLE 中断能正常分帧
- 实际上串口 `send()` 之间总有数毫秒调度延迟，无需刻意处理

**其它约束：**
- 单帧最大长度 32 字节
- 未识别的 tag / 数值非法 / 数值 > 255 → 收方忽略（不 NACK）

## 二、命令表

### MCU → ASRPRO

| 帧文本            | 语义         | 说明                        |
| ---------------- | ------------ | --------------------------- |
| `play=<id>`      | 播报预置语音 | id = 语音 ID（见第三节）   |
| `stop`           | 停止当前播报 | —                           |
| `ping`           | 心跳/存活探测 | —                           |

### ASRPRO → MCU

| 帧文本            | 语义         | 说明                        |
| ---------------- | ------------ | --------------------------- |
| `cmd:<id>`       | 语音命令识别 | id = 命令 ID（见第四节）   |
| `wake`           | 检测到唤醒词 | —                           |
| `done:<id>`      | 播报完成     | id = 上次的语音 ID          |

## 三、语音 ID 表（十进制）

```
01 开机语     02 关机语     03 进入动力
04 进入感应   05 进入遥控   06 进入语音
07 遥控连接   08 遥控断开   09 低电量
10 收到       11 前进 12 后退 13 左转 14 右转 15 停止
20 靠近启动   21 遇障停止   22 挥手开关   23 明暗调速
```

## 四、命令 ID 表（十进制）

```
30 前进 31 后退 32 左转 33 右转 34 停止
40 左电机正转 41 左电机反转 42 左电机停止
43 右电机正转 44 右电机反转 45 右电机停止
50 进入动力 51 进入感应 52 进入遥控 53 进入语音
```

## 五、示例

**MCU 请求播开机语：** 发送 7 字节 `play=01`（hex: `70 6C 61 79 3D 30 31`）

**ASRPRO 识别到"前进"：** 发送 6 字节 `cmd:30`

**ASRPRO 上报开机语已播完：** 发送 7 字节 `done:01`

## 六、变更记录

- **v0.4 (2026-07-03)**：取消 `\r\n` 帧尾。原因：ASRPRO 端图形化工具处理字符串拼接控制字符不方便，改为 UART IDLE 分帧简化两端实现。
- **v0.3 (2026-07-03)**：不对称分隔符（发 `=`、收 `:`）+ 数值统一 dec。
- **v0.2 (2026-07-03)**：ASCII 文本、冒号分隔、hex 数值。已废弃。
- **v0.1 (2026-07-03)**：定长 6 字节二进制帧。已废弃。
