# 小白-Ai MCU ↔ ASRPRO UART 交互协议 v0.2

- 更新日期：2026-07-03（v0.2：从定长 6 字节二进制帧改为 ASCII 文本帧）
- 物理层：UART 115200 8N1 全双工
- 目标 MCU：PY32F030K28U6TR
- 语音芯片：ASRPRO
- 最大帧长：**32 字节**（含 `\r\n`）

---

## 一、帧格式

ASCII 文本，`\r\n` (0x0D 0x0A) 作为帧结束符。

```
<tag>[:<hex>]\r\n
```

- `tag`：全小写英文字母，长度 ≤ 8（例如 `play`、`cmd`、`wake`、`done`、`stop`、`ping`）
- `hex`（可选）：2 位十六进制数值，大小写均可（例如 `30`、`3A`、`0a`）
- 结束符：`\r\n` 两字节

**约束：**
- 收方在遇到 `\r\n` 时才判定为一个完整帧
- 单帧总长超过 32 字节 → 收方直接丢弃当前缓冲，重新开始收帧
- 未识别的 tag、hex 位数不对、格式非法 → 收方忽略该帧（不 NACK 不响应）
- UART 8N1 无奇偶校验，靠 `\r\n` 分帧和 tag 白名单容错

## 二、命令表

### MCU → ASRPRO

| 帧文本            | 语义         | 说明                        |
| ---------------- | ------------ | --------------------------- |
| `play:XX\r\n`    | 播报预置语音 | XX = 语音 ID（见第三节）   |
| `stop\r\n`       | 停止当前播报 | —                           |
| `ping\r\n`       | 心跳/存活探测 | —                           |

### ASRPRO → MCU

| 帧文本            | 语义         | 说明                        |
| ---------------- | ------------ | --------------------------- |
| `cmd:XX\r\n`     | 语音命令识别 | XX = 命令 ID（见第四节）   |
| `wake\r\n`       | 检测到唤醒词 | —                           |
| `done:XX\r\n`    | 播报完成     | XX = 上次的语音 ID          |

## 三、语音 ID 表

```
01 开机语     02 关机语     03 进入动力
04 进入感应   05 进入遥控   06 进入语音
07 遥控连接   08 遥控断开   09 低电量
0A 收到       10..14 前进/后退/左转/右转/停止
20..23 靠近启动/遇障停止/挥手开关/明暗调速
```

## 四、命令 ID 表

```
30 前进 31 后退 32 左转 33 右转 34 停止
40 左电机正转 41 左电机反转 42 左电机停止
43 右电机正转 44 右电机反转 45 右电机停止
50 进入动力 51 进入感应 52 进入遥控 53 进入语音
60 进入风扇玩法 61 打开风扇 62 关闭风扇
70 进入投石机玩法 71 发射
```

## 五、示例

**MCU 请求播开机语：**
```
play:01\r\n
```
（hex: `70 6C 61 79 3A 30 31 0D 0A`）

**ASRPRO 识别到"前进"：**
```
cmd:30\r\n
```

**ASRPRO 上报开机语已播完：**
```
done:01\r\n
```

## 六、变更记录

- **v0.2 (2026-07-03)**：从定长 6 字节二进制帧（`A5 03 CMD D0 D1 XOR`）改为 ASCII 文本帧。原因：ASRPRO 端图形化开发工具处理文本更方便，PC 串口助手 ASCII 模式可直接读。
- **v0.1 (2026-07-03)**：初版二进制协议，已废弃。
