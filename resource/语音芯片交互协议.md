# 小白-Ai MCU ↔ ASRPRO UART 交互协议 v0.6

- 更新日期：2026-07-09（v0.6：MCU→ASR 方向新增帧尾 `\n`,两方向统一带帧尾,根治语音侧按行分帧的粘包/超时问题）
- 物理层：UART 9600 8N1 全双工
- 目标 MCU：PY32F030K28U6TR
- 语音芯片：ASRPRO
- 最大帧长：**32 字节**（含帧尾）

---

## 一、帧格式

ASCII 文本。两个方向都用 **`=`** 作为 tag 与数值的分隔符,且**都带帧尾**。

**MCU → ASRPRO：** 帧尾 `\n`（LF 0x0A）
```
<tag>[=<dec>]\n
```

**ASRPRO → MCU：** 帧尾 `\r\n`（CR 0x0D + LF 0x0A）
```
<tag>[=<dec>]\r\n
```

- `tag`：全小写英文字母,长度 ≤ 8（例如 `play` / `cmd` / `wake` / `done` / `stop` / `ping`）
- `dec`：十进制数值字符串（1..3 位,`1`、`01`、`10`、`100` 均接受,值 ≤ 255）
- 收方在遇到帧尾（MCU→ASR 的 `\n` 或 ASR→MCU 的 `\r\n`）时判定为一个完整帧

**为什么两方向都带帧尾?**
- v0.6 起两方向统一带帧尾。语音端图形化工具（天问/ASRPRO）用 `serial_readstr` 按行读取,依赖帧尾 `\n` 做**确定性分帧**——无帧尾时它只能靠字符间静默超时断帧,会导致粘包（如 `play=06play=11`）和额外延迟。加帧尾后这些问题被根治。
- MCU 端接收用 UART DMA + IDLE 中断,兼容有无帧尾（收到 `\r\n` 或 `\n` 均正确剥尾）。

**其它约束:**
- 单帧超 32 字节 → 收方丢弃
- 未识别 tag、格式非法、数值 > 255 → 收方静默忽略

## 二、命令表

### MCU → ASRPRO（发送时带 `\n` 尾）

| 帧文本            | 语义         | 说明                        |
| ---------------- | ------------ | --------------------------- |
| `play=<id>`      | 播报预置语音 | id = 语音 ID（见第三节）   |
| `stop`           | 停止当前播报 | —                           |
| `ping`           | 心跳/存活探测 | —                           |

### ASRPRO → MCU（发送时带 `\r\n`）

| 帧文本               | 语义         | 说明                        |
| ------------------- | ------------ | --------------------------- |
| `cmd=<id>\r\n`      | 语音命令识别 | id = 命令 ID（见第四节）   |
| `wake\r\n`          | 检测到唤醒词 | —                           |
| `done=<id>\r\n`     | 播报完成     | id = 上次播报的语音 ID     |

## 三、语音 ID 表（十进制）

```
01 开机语     02 关机语     03 进入动力
04 进入感应   05 进入遥控   06 进入语音
07 遥控连接   08 遥控断开   09 低电量
10 收到       11 前进 12 后退 13 左转 14 右转 15 停止
20 靠近启动   21 遇障停止   22 挥手开关   23 明暗调速
```

## 四、命令 ID 表（十进制）

```
30 前进 31 后退 32 左转 33 右转 34 停止
40 左电机正转 41 左电机反转 42 左电机停止
43 右电机正转 44 右电机反转 45 右电机停止
50 进入动力 51 进入感应 52 进入遥控 53 进入语音
```

## 五、示例

**MCU 请求播开机语:** 发送 8 字节 `play=01\n`（hex: `70 6C 61 79 3D 30 31 0A`）

**ASRPRO 识别到"前进":** 发送 8 字节 `cmd=30\r\n`（hex: `63 6D 64 3D 33 30 0D 0A`）

**ASRPRO 上报开机语播报完成:** 发送 9 字节 `done=01\r\n`

**ASRPRO 检测到唤醒词:** 发送 6 字节 `wake\r\n`

## 六、变更记录

- **v0.6 (2026-07-09)**:MCU→ASR 新增帧尾 `\n`;两方向统一带帧尾。语音侧改按行确定性分帧,MCU 侧删除播报后 500ms 阻塞延时。根治"播报丢失/不同步/传输慢"。
- **v0.5 (2026-07-03)**:两方向统一 `=` 分隔（去掉 v0.3/v0.4 的 `:`）;MCU→ASR 保持无帧尾（IDLE 分帧）,ASR→MCU 带 `\r\n`。
- **v0.4 (2026-07-03)**:取消所有帧尾;MCU→ASR 用 `=`,ASR→MCU 用 `:`。
- **v0.3 (2026-07-03)**:不对称分隔符 `=` / `:`;数值统一 dec;带 `\r\n`。
- **v0.2 (2026-07-03)**:ASCII 文本、冒号分隔、hex 数值。已废弃。
- **v0.1 (2026-07-03)**:定长 6 字节二进制帧。已废弃。
